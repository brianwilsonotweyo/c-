Chapter 1: Understanding C++ Syntax

In this module we will dissect the code from the previous lesson and go through it step by step in order to understand what exactly the anatomy of a C++ program/ file comprises of ?

Code snippet: Baseline C++ code:

#include using namespace std; int main() { cout << "Hello World!"; return 0; }

  • In the [#include ] we are basically importing the iostream library that comes inbuilt with C++ by default. And in this case this library [iostream] lets us work with inputting and outputting objects, such as cout (used in line 5). Imports are also called Header files in C++ and they add functionality to C++ programs as just with our code alone we can only do so little compared to the libraries that come already shipped with C++.

  • In the [using namespace std] , this basically means that we can use names for objects and variables from the standard library of C++.

    • Note: The [#include ] and [using namespace std] are by default something that (almost) always appears in a C++ program thus there is no need to keep ranting about what it means as by default they'll always be in our code for it to run.

  • In the [int main()] part of our code, int is called a function in C++ and any code inside its curly brackets curly braces '{}' will be executed. thus all code in the curly braces of a function is always executed

  • In the [cout << "Hello World!";] usually pronounced "see-out" the cout is an object used together with the insertion operator (<<) to output/print text. In our example it will output "Hello World".

  • In the  [return 0]  is where the main function ends .

    • Note:

    • Every C++ statement ends with a semicolon ;.
    • The C++ compiler ignores blank spaces in our code .
